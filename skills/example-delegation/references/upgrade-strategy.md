# 升级策略详细参考

> 此文件为 L3 参考，仅在 agent 遇到用户持续不满意或任务执行循环时需要查阅。

## 升级决策树

```
任务执行
  │
  ├─ 用户满意？ → 是 → 完成
  │
  └─ 否 → 第几轮修正？
            │
            ├─ 第1轮 → 理解反馈，调整方案重试
            │
            ├─ 第2轮 → 检查：是否在同一思路上变体？
            │           │
            │           ├─ 是 → 升级
            │           └─ 否 → 换思路重试
            │
            └─ ≥3轮 → 强制升级，不再重试
```

## 升级信息包模板

```
【升级请求】
原始需求: {用户最初的需求描述}
反馈历程:
  - Round 1: {用户反馈} → 尝试了 {方案} → 失败原因: {原因}
  - Round 2: {用户反馈} → 尝试了 {方案} → 失败原因: {原因}
当前阻塞点: {具体卡在哪里}
已排除假设:
  - {假设1}: 已确认不成立，因为 {证据}
  - {假设2}: 已确认不成立，因为 {证据}
代码状态: {当前代码的关键状态描述}
```

## 升级后的处理

升级模型收到信息包后应：
1. **不立即实现** — 先重新诊断根因
2. **质疑假设** — 之前排除的假设是否真的排除了？
3. **改变维度** — 从不同角度拆解问题
4. **提出新方案** — 基于新的根因分析
