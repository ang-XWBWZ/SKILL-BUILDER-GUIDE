# 八种技能类型完整目录

## 技能类型与双轴默认值

| # | 类型 | 用途 | 默认执行层 | 默认组合层 | 典型问题 |
|---|------|------|:--:|:--:|---------|
| 1 | 规范技能 (dev) | 技术栈、代码规范、API规范 | L1 | atomic | "用什么技术？怎么组织？" |
| 2 | 地图技能 (code-map) | 文件定位、组件导航 | **L0** | atomic | "在哪写？哪个文件？" |
| 3 | 流程技能 (workflow) | 开发步骤、检查清单 | L1 | functional | "按什么顺序？从哪开始？" |
| 4 | 脚本技能 (scripts) | 部署回滚、日常运维 | L0 | atomic | "怎么运行？怎么部署？" |
| 5 | 调用链技能 (call-chain) | 数据流追踪、类型验证 | L1 | functional | "数据怎么传？终点在哪？" |
| 6 | 变更日志技能 (diffs) | 变更记录、历史追踪 | L1 | functional | "改了什么？为什么改？" |
| 7 | 变更模型技能 (change-model) | 结构化变更报告 | L1 | functional | "为什么改？影响什么？怎么验证？" |
| 8 | 分治技能 (delegation) | 模型路由、L0强制下放 | L1 | planning | "谁来做？该不该派Haiku？" |

## 数量决策矩阵

| 项目特征 | 推荐数量 | 技能组合 |
|---------|:------:|---------|
| 小型 (Bug修复、CSS调整) | 2 | 规范(atomic) + 流程(functional) |
| 中型 (新增页面/组件) | 3 | 规范 + 地图(atomic) + 流程 |
| 大型 (新增模块/改架构) | 4 | 规范 + 地图 + 流程 + 变更模型(functional) |
| 复杂 (多服务、多团队) | 5-6 | 以上 + 调用链(functional) + 脚本(atomic) + 分治(planning) |

## 按场景推荐

| 场景 | 推荐技能 | 原因 |
|------|----------|------|
| 新人入职 | 地图 + 流程 | 快速了解项目结构 |
| 日常开发 | 规范 + 地图 | 查规范、定位文件 |
| 变更发布 | 变更模型 + 调用链 | 结构化变更报告 |
| 问题排查 | 调用链 + 规范 | 追踪数据流 |
| 性能优化 | 规范 + 调用链 | 定位瓶颈 |

## 技能职责边界

| 技能类型 | 职责边界 | 核心价值 |
|---------|---------|---------|
| 规范技能 | 标准规范 | 技术选型、代码组织 |
| 地图技能 | 代码位置 | 文件导航、组件定位 |
| 流程技能 | 开发步骤 | 流程引导、检查清单 |
| 日志技能 | 变更记录 | 追踪历史、复盘分析 |
| 调用链技能 | 数据流追踪 | 调用关系、类型验证 |
| 脚本技能 | 运行维护 | 部署回滚、日常运维 |
| 分治驱动 | 模型编排 | 强制L0下放、模型路由 |
| 变更模型 | 变更文档 | 因果链追踪 |

## 创建时机判断

| 场景 | 判断标准 | 应创建的新技能 |
|------|---------|---------------|
| 开发流程有缺失环节 | 开发流程中没有覆盖的步骤 | 如"调用链检查" |
| 重复性问题需系统化 | 同一类问题出现 3 次以上 | 根据问题类型 |
| 新增独立工具/系统 | 项目引入新模块 | 如"监控系统" |
| 需要固化最佳实践 | 团队成员反复询问同一类问题 | 根据问题类型 |
