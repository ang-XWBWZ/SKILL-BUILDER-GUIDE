# 变更存档设计（完整方法论）

> 对应 `SKILL-BUILDER-GUIDE.md` 第八章。此文件为 L3 参考，仅在 agent 需要设计或修改存档方案时加载。

## 为什么需要存档

| 价值 | 说明 |
|------|------|
| **可追溯** | 按日期索引，随时查看任何时间段的变更历史 |
| **可复盘** | 出问题时回溯"当时为什么这样改" |
| **AI 上下文** | 后续 AI 会话可读取历史报告，理解项目演进脉络 |
| **团队对齐** | 新人或协作者快速了解近期变动 |

## 存档时机

```
编码完成 → 调用链检查 → 测试验证 → 生成最终报告 → 📦 归档
```

| 阶段 | 操作 | 说明 |
|------|------|------|
| 变更前 | 创建报告草稿 | 填写 WHY / WHAT 部分 |
| 编码完成 | 填写 HOW 部分 | 实施完成后的文件清单和 diff |
| 测试前 | 填写 VALIDATION（调用链检查） | 确保链路正确 |
| 测试通过 | 完善测试结果 | 记录实际测试数据 |
| **归档** | **执行存档命令** | **最终版本落地到 docs/changes/** |

## 目录结构

```
{project}/
└── docs/
    └── changes/                  # 变更报告存档目录
        ├── INDEX.md              # 总索引（按时间 / 按类型双维度排列）
        ├── 2026-04-26/           # 按日期组织
        │   ├── 20260426-01-新增筛选功能.md
        │   └── 20260426-02-修复登录超时.md
        └── 2026-04-27/
            └── 20260427-01-重构支付模块.md
```

## 命名规范

```
{YYYYMMDD}-{NN}-{简短描述}.md

YYYYMMDD  — 归档日期（8位）
NN        — 当天序号（2位，从01开始，自动递增）
简短描述  — 功能名/变更主题（建议20字以内，英文用中划线连接）
```

## 索引维护守则

- **追加不覆盖** — 新条目追加，不覆盖已有记录
- **序号连续** — 同一天内的报告序号从 01 递增
- **类型准确** — 如实标记 feature / bugfix / refactor / config
- **状态更新** — 报告归档时默认 DONE，后续如需废弃可改为 DEPRECATED
